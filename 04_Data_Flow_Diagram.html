Data Flow Diagram (DFD)
Visual map of how personal data moves through your organization
Purpose: Create a visual representation of how personal data flows from collection through processing to deletion. This helps identify gaps in security and compliance.

Instructions

  1. Map each touchpoint where personal data is collected (web, mobile, paper, API, import, etc.)
  2. Show where data is stored (databases, files, caches, backups)
  3. Document all processors/vendors who receive data
  4. Show where data is processed (anonymization, analytics, reporting)
  5. Identify retention points and deletion triggers
  6. Create visual diagram (can be ASCII, flowchart, or attached image)
