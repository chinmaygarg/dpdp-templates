Compliance Audit Plan
17-Domain Compliance Verification Methodology
Version: 1.0 | Effective Date: [DATE] | Status: [Active/Draft/Archived]

1. Purpose & Scope (Phase 2: Gap Analysis)

Objective: Conduct a comprehensive compliance audit against all 92 DPDP checkpoints across 17 domains. Document control effectiveness, identify gaps, and prioritize remediation.

Scope: Phase 2 audit executes 17-domain assessment (Sections 5-16 of DPDP Act) covering consent, security, data rights, vendor management, breach response, and governance.

Output: Audit Findings Report (T-28) with gap severity scoring (CRITICAL/HIGH/MEDIUM/LOW) and remediation roadmap.

2. 92-Checkpoint Framework (17 Domains)

Domain Checkpoints Key DPDP Sections Audit Focus
1. Consent & Notice 8 Sec 6 (FSIUU) 5-attribute FSIUU scoring, privacy notice version control
2. Data Subject Rights 7 Sec 11-14 (DSAR/Correction/Erasure/Nomination) SLA enforcement (30d DSAR, 15d correction, 30d erasure)
3. Legitimate Uses 6 Sec 7 (9 categories) Use case documentation, CPaaS channel compliance
4. Information Security 12 Sec 8 (data security controls) Encryption, MFA, access control, SIEM
5. Children's Data 4 Sec 9 (parental consent, data minimization) Age verification, parental consent records
6. DPIA & DPO 5 Sec 10 (DPIA triggers, DPO independence) DPIA register, DPO reporting line
7. Breach Notification 5 Sec 20 (72-hr DPBI, 6-hr CERT-In) SLA compliance, incident response process
8. Retention & Deletion 6 Sec 17 (30-day deletion SLA) Deletion log, processor confirmation
9. Vendor & Processor 7 Sec 12(4) (DPA, sub-processor control) DPA signatures, audit rights, propagation SLA
10. Cross-Border Transfers 4 Sec 16 (adequacy + explicit consent) Transfer register, adequacy assessment
11. Audit & Monitoring 5 Sec 8 (audit trail, access logs) Hash-chained logs, SIEM alerting, data discovery
12. Grievance Handling 4 Sec 15 (30-day SLA) Grievance register, SLA tracking, escalation
13. Withdrawal 3 Sec 6(4) (no friction, 24-hr propagation) Withdrawal log, propagation SLA, no friction control
14. Legitimate Use Documentation 2 Sec 7 (Section 7 memos) Written justification per use case
15. Privacy by Design 3 Sec 5 (accountability, data minimization) Built-in DSAR/deletion, privacy impact analysis
16. Employee Training 2 Sec 5 (awareness, role-specific) Training completion rate ≥95%
17. Governance & Board 3 Sec 10 (Privacy Committee, Board oversight) Board resolution, quarterly risk reports

3. Audit Methodology (Testing Approach)

A. Document Review (Weeks 1-2):
1. Privacy policies, consent forms, notice templates → FSIUU compliance scoring
2. Data Processing Agreement signatures → 100% processor coverage
3. ROPA entries → all active processing justified
4. Section 7 use case memos → 9 categories covered

B. System Testing (Weeks 2-4):
1. Encryption verification → AES-256-GCM in transit + rest, TLS 1.3+
2. Access control audit → quarterly review evidence, MFA enforcement
3. DSAR workflow → 30-day SLA compliance on sample of 20 requests
4. Deletion execution → 30-day SLA + processor confirmation (Section 12(4))
5. Breach response simulation → 6-hour CERT-In, 72-hour DPBI readiness
6. Data discovery scan → PII masking compliance, shadow data detection

C. Control Walkthrough (Weeks 1-4):
1. Consent withdrawal path → no friction, real-time processing
2. Employee access path → RBAC enforcement, 2-hour JIT escalation
3. Audit log access → immutability, hash-chaining, SIEM alerting
4. Incident response → playbook execution, SLA monitoring
5. Processor escalation → Section 12(4) confirmation workflow

5. Audit Schedule & Roles

Role Responsibility Qualifications
Lead Auditor Overall audit execution, sampling decisions, findings determination DPDP Act expertise, audit experience, independent of operations
Technical Auditor System testing (encryption, access, logs), security controls verification Security engineering, database knowledge, cryptography basics
Legal Auditor Policy review, DPA assessment, consent form FSIUU validation Data protection law, contract review, regulatory background

6. Audit Timeline (Weeks 1-4)

Week 1: Kickoff + document review (consent, DPA, ROPA, policies)
Week 2: System testing begins (encryption, access control, logs)
Week 3: Control walkthrough + SLA sampling (DSAR, deletion, breach)
Week 4: Findings aggregation, severity scoring, remediation roadmap draft, management review

Post-Audit (Week 5): Board presentation, DPO sign-off, Audit Findings Report (T-28) publication

7. Compliance Scoring Formula

Overall Compliance Score = (Checkpoints Passing / 92) × 100%

Per-Domain Score = (Checkpoints Passing / Domain Checkpoints) × 100%

Checkpoint Severity Weighting (for prioritization, not score calculation):
• CRITICAL (missing mandatory control): ₹50-100 Cr penalty, 30-day remediation SLA
• HIGH (incomplete mandatory): ₹25-50 Cr penalty, 60-day remediation SLA
• MEDIUM (minor gap): ₹10-25 Cr penalty, 90-day remediation SLA
• LOW (non-material): <₹10 Cr penalty, no deadline

8. Implementation Checklist

  • ☐ Establish Audit Committee (DPO, Legal, Compliance, one Board member)
  • ☐ Hire or contract independent auditor (external if budget allows)
  • ☐ Define audit scope: all 92 checkpoints, 17 domains (mandatory)
  • ☐ Set audit start date, 4-week execution timeline
  • ☐ Prepare access for auditors (systems, data, logs, document access)
  • ☐ Freeze code deployments during audit week 1 (stable baseline)
  • ☐ Extract evidence documentation: RoPA, DPAs, consent logs, deletion logs, breach logs
  • ☐ Create sample datasets (50 consents, 20 DSAR, 30 deletions, 100 access events)
  • ☐ Configure audit trail access (hash-chained logs readable to auditors)
  • ☐ Schedule walkthroughs (DSAR flow, deletion flow, breach simulation, access control demo)
  • ☐ Weekly auditor sync-up (blockers, evidence gaps, findings discussion)
  • ☐ Collect audit findings on weekly basis (not last minute Week 4 crunch)
  • ☐ Draft Audit Findings Report (T-28) with gap register
  • ☐ Management review findings before external disclosure
  • ☐ DPO final review and sign-off on audit scope, methodology, findings
  • ☐ Board presentation of Audit Findings Report + remediation roadmap
  • ☐ Publish findings, secure 7-year retention of all audit documentation
