Data Subject Access Request (DSAR) Response (DPDP Section 11)
Version: 2.0 (Updated April 2026) | Effective Date: 13 May 2027 | Status: Active

Overview

Under DPDP Act 2023 Section 11, Data Principals have the unconditional right to request access to their personal data. Organizations must respond within the statutory SLA with an automated acknowledgement, a complete data export, and clear information about portability rights and escalation options.

6 Data Principal Rights (Section 11 + Related Sections)

DSAR is not just "access to data". The DPDP Act grants Data Principals 6 distinct rights, all of which must be addressed in the DSAR response or in linked templates.

Right DPDP Section What They Get SLA
1. Access Section 11 Complete copy of personal data in structured format (CSV, JSON, XML) 30 days
2. Portability Section 13 Data in machine-readable, commonly-used format transferable to another organization 30 days
3. Correction Section 12 Ability to correct inaccurate/incomplete data (separate form/link provided) 30 days to complete correction
4. Erasure Section 12 Deletion of data (with limited exceptions for legal obligations, ongoing processing) 30 days to complete erasure
5. Nomination Section 14 Designate nominee to access data post-death (form linked in response) Ongoing right, managed via portal
6. Grievance Section 15 File grievance if response is incomplete/unsatisfactory; escalate to DPBI 60 days DPO response

Identity Verification Process

Before disclosing personal data, organization must verify requester's identity to prevent unauthorized data access. Acceptable methods depend on the data and organization's existing authentication:

Method Strength Process Delay
Account Login + OTP High (if existing account) Portal login with 2FA; OTP sent to registered email/SMS <5 min
Aadhaar VID Very High UIDAI-issued VID (Virtual ID) authenticated via AADHAAR API; no 12-digit Aadhaar stored 1-5 min
Email + Callback Verification Medium DSAR submitter confirms control of email; verify requestor phone number on file 1-24 hours
PAN + DOB Verification Medium Last 4 digits of PAN + Date of Birth match on file (masked in all logs to last 4 digits) <10 min
Video KYC Highest Live video call with ID document verification; Liveness check 15-30 min
✅ BEST PRACTICE:
Multi-factor approach: Combine account login (existing) + OTP (frictionless) + optional VID for high-value data requests. Document verification method in DSAR log for audit trail.

DPBI Escalation Path

If a Data Principal is unsatisfied with the organization's response to a DSAR or related request (correction, erasure, grievance), they can escalate to the Data Protection Board of India (DPBI) — the statutory regulator.

DPBI Escalation Timeline

  • Within 60 days of DSAR response: Data Principal must file complaint with DPBI (link: [DPBI_COMPLAINT_PORTAL])
  • DPBI Investigation: DPBI investigates complaint and may issue orders
  • DPBI Orders are binding: Organization must comply with DPBI decision
  • Penalties for non-compliance: Up to ₹50 Cr or 2% annual revenue (whichever is higher) under Section 25
⚠️ CRITICAL:
Organizations must take DPBI escalation seriously. Failure to respond to a DPBI order can result in penalties up to ₹50 Cr or 2% of annual revenue. Always complete DSAR within the 30-day SLA to avoid escalation.

Implementation Checklist

  • ☐ Process: DSAR intake form created on portal (email + web form both supported)
  • ☐ Automation: 48-hour auto-ACK email configured with request ID, expected completion date, tracking link
  • ☐ Identity Verification: Multi-factor verification process documented (account login + OTP, VID, callback, etc.)
  • ☐ Data Extraction: SQL queries or ETL scripts prepared to extract all personal data per Data Principal
  • ☐ Data Formatting: Export pipeline creates CSV, JSON, and PDF formats; encryption in transit
  • ☐ Portability: Data provided in machine-readable format (CSV/JSON compatible with other tools)
  • ☐ Correction Link: Response letter includes link to T-12 (Correction Form)
  • ☐ Erasure Link: Response letter includes link to T-12 (Erasure Request Form)
  • ☐ Nomination Link: Response letter includes link to T-14 (Nomination Form)
  • ☐ Grievance Link: Response letter includes link to T-09 (Grievance Form) + DPO contact
  • ☐ DPBI Info: Response letter includes DPBI escalation information and complaint portal link
  • ☐ Logging: DSAR tracker table records request ID, submission date, verification date, response date, compliance status
  • ☐ Audit Trail: All DSAR actions (receipt, ACK, verification, extraction, response) logged in hash-chained audit log
  • ☐ SLA Monitoring: Dashboard tracks all DSARs, flags those approaching 30-day deadline
  • ☐ Data Retention: Downloaded data on server encrypted and purged after 90 days
  • ☐ Legal Review: Response letter template approved by Legal + DPO
  • ☐ Testing: End-to-end DSAR flow tested: submit → ACK → verify identity → extract → deliver → verify portability
  • ☐ Training: Grievance team trained on DSAR process, SLA, escalation path
