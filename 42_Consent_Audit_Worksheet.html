FSIUU = Free, Specific, Informed, Unconditional, Unambiguous (DPDP Act Section 6)
Audit Instructions: For each consent record sampled, verify ALL 5 elements against the criteria below. Mark PASS, FAIL, or ? UNCLEAR.
ELEMENT 1: FREE (No Coercion or Bundling)
Criterion PASS FAIL UNCLEAR ? Evidence / Notes
No pre-ticked boxes (affirmative action required by user)
Consent not bundled with service access (user can refuse)
No dark patterns or deceptive interface design
Withdrawal is as easy as granting consent
ELEMENT 2: SPECIFIC (Per-Purpose, Not Blanket)
Criterion PASS FAIL UNCLEAR ? Evidence / Notes
Consent obtained for EACH processing purpose separately
No blanket "consent to all processing" language
Each purpose clearly distinguishable in consent form
Secondary / new purposes require explicit re-consent
ELEMENT 3: INFORMED (Privacy Notice Given Before Consent)
Criterion PASS FAIL UNCLEAR ? Evidence / Notes
Privacy notice is STANDALONE document (not buried in T&Cs)
Notice provided BEFORE or AT TIME of consent
Notice lists EACH item of personal data being collected
Notice explains DATA PRINCIPAL RIGHTS (access, correct, erase, etc.)
Notice in appropriate language (English or regional as applicable)
ELEMENT 4: UNCONDITIONAL (No Coercive Bundling)
Criterion PASS FAIL UNCLEAR ? Evidence / Notes
User can refuse optional consent without losing core service
Non-compliance penalty (e.g., higher price) not applied for refusal
Legitimate uses (Section 7) not falsely labeled as "consent"
ELEMENT 5: UNAMBIGUOUS (Clear, Affirmative, Recorded)
Criterion PASS FAIL UNCLEAR ? Evidence / Notes
Consent captured by explicit user action (click, sign, verbal recorded)
Consent RECORDED with timestamp, user ID, version of notice
Consent remains unaltered from point of capture to audit date
Audit trail of consent available (timestamp, IP, device info)
