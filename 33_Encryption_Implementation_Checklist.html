1. Purpose & Scope

This checklist verifies the implementation of encryption controls for protecting personal data at rest and in transit, in compliance with DPDP Act 2023 and industry security standards (ISO 27001, NIST).

4. Key Management Controls

Key Management Requirements:

  • Keys stored separately from encrypted data
  • Key rotation at least annually
  • Multi-factor authentication for key access
  • Complete audit trail of key usage
  • Secure key destruction procedures

6. India-Specific Regulatory Encryption Rules

These rules are mandatory for all India-regulated organizations and create direct penalty exposure if violated.

Rule 1: Aadhaar Virtual ID (VID) Only — NO 12-Digit Aadhaar Storage

✗ VIOLATION: Storing 12-digit Aadhaar number in database, logs, or backups
✓ COMPLIANCE: Store only Aadhaar Virtual ID (VID) from UIDAI

Database Encryption Impact:
• If 12-digit Aadhaar exists in DB: Immediate remediation required (delete plaintext, replace with hashed + VID)
• If VID is stored: Encryption at rest is sufficient (AES-256-GCM column-level)
• Hash Aadhaar for deduplication: SHA-256(12-digit Aadhaar) — store hash only, never plaintext

Rule 2: PAN Masking in All Outputs (Except Payment APIs)

PAN Storage vs. Display Rule:

In Database: Full PAN stored, encrypted with AES-256-GCM at rest
In All Outputs: Mask to last 4 digits (e.g., XXXXXXX1234)

Applies to:
✓ Admin dashboards
✓ API responses (JSON)
✓ Exports (CSV, Excel, PDF)
✓ Logs (application, audit, SIEM)
✓ Error messages and exceptions
✓ Email communications
✓ Backups and archives

Exception: Full PAN only in payment processing APIs (with restricted access + audit logging)

Sensitive Data Field Registry

Audit your database for these Tier 1 PII fields. Each requires specific masking/encryption rules:

PAN 10-character string Encrypted AES-256-GCM Mask to XXXXXXX1234 (last 4 only) ☐ Verified

Aadhaar & PAN Encryption Verification Checklist

  • ☐ Audit database: Search all tables for 12-digit Aadhaar column pattern (XXXXXXXX****). MUST be zero matches.
  • ☐ Verify VID storage: Aadhaar-like fields should contain 16-digit VID, encrypted at column level.
  • ☐ PAN masking in APIs: All API responses with PAN show XXXXXXX1234 format. Full PAN encrypted in DB only.
  • ☐ PAN masking in logs: Grep application logs for PAN patterns. Any matches must be redacted as [PAN-REDACTED].
  • ☐ Audit exports: Generate sample CSV/Excel/PDF exports. Verify PAN and Aadhaar are masked/redacted.
  • ☐ Audit backups: Database backups are encrypted (AES-256) and include only VID (not 12-digit Aadhaar).
  • ☐ Email communications: Data shared via email has Aadhaar redacted and PAN masked.
  • ☐ Exception documentation: Document any fields where full PAN/Aadhaar is displayed (must be rare + approved by DPO).
