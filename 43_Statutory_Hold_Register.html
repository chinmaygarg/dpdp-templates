CRITICAL NOTE: When a data subject requests erasure (Section 8(7)) but a statutory hold applies, the organization must DENY or DEFER the request and inform the principal of the legal basis and hold expiry date. This register tracks all data types subject to statutory holds and the applicable retention periods.

ERASURE REQUEST HANDLING PROTOCOL
When a data principal files an erasure request:
1. Check this register for any applicable statutory hold
2. If hold applies: DENY + send principal written explanation with hold end date
3. Log rejection in DSAR system with hold reason + expiry date
4. If no hold: Process erasure per DPDP Section 8(7) SOP
5. If hold expires during pendency: Re-evaluate request and proceed with deletion
