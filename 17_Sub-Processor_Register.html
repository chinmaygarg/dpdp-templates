1. Purpose & Legal Basis

This register maintains a record of all sub-processors engaged by Data Processors on behalf of the Data Fiduciary, in compliance with DPDP Act 2023. Under Section 8, Data Processors must not engage another processor without prior written consent of the Data Fiduciary.

2. Definitions

Term Definition
Data Fiduciary Entity that determines the purpose and means of processing personal data
Data Processor Entity that processes personal data on behalf of the Data Fiduciary
Sub-Processor Third party engaged by the Data Processor to process personal data
Processing Collection, recording, organization, structuring, storage, adaptation, retrieval

4. Approval Process

Approval Workflow:

  • Data Processor submits sub-processor request with details
  • Data Fiduciary reviews security measures and DPA terms
  • Legal/Compliance reviews data transfer mechanisms (if cross-border)
  • Written approval granted or denied
  • Sub-Processor added to this register upon approval

5. Security Requirements

All Sub-Processors must demonstrate:

  • ISO 27001 certification or equivalent
  • AES-256 encryption for data at rest
  • TLS 1.2+ for data in transit
  • Role-based access controls
  • Incident response procedures
  • Data retention and deletion policies
  • Regular security audits
  • Employee training on data protection

7. Obligations of Sub-Processors

Each sub-processor must:

  • Process personal data only on documented instructions from the Data Processor
  • Ensure confidentiality of employees with access to personal data
  • Implement appropriate technical and organizational security measures
  • Assist the Data Processor in fulfilling its obligations to Data Principals
  • Delete or return all personal data upon termination of the agreement
  • Make available information necessary to demonstrate compliance
  • Permit and cooperate with audits conducted by the Data Fiduciary

8. Notification Requirements

Critical: The Data Processor must notify the Data Fiduciary immediately if:

  • A sub-processor is engaged without prior approval
  • A security breach occurs at the sub-processor level
  • The sub-processor intends to sub-delegate to another entity
  • There is a material change in the sub-processor's security posture
